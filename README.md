# Automated-Testing-Framework-for-E-Commerce-Website-Using-Selenium-and-TestNG
Manual testing of e-commerce applications is time-consuming, repetitive, and prone to human error. This project solves these issues by implementing a reusable automated testing framework using Selenium and TestNG.
2. Project Idea / Introduction
This project focuses on developing an automated software testing framework for an e-commerce website
using Selenium WebDriver and TestNG. The testing system verifies functionalities such as user
registration, login, logout, add to cart, remove from cart, and guest user operations.
3. Problem Statement
Manual testing of e-commerce applications is time-consuming, repetitive, and prone to human error. This
project solves these issues by implementing a reusable automated testing framework using Selenium
and TestNG.
4. Objectives
• Automate testing of major e-commerce functionalities.
• Validate user registration and login processes.
• Test cart management operations.
• Perform functional, smoke, UI, end-to-end, and cross-browser testing.
• Improve software quality.
5. Features / Functionalities
Registration Module: Registration validation and security checks.
Login Module: Login verification and credential handling.
Logout Module: Session invalidation and logout validation.
Add to Cart: Product addition, persistence, and guest handling.
Remove from Cart: Item deletion and empty cart validation.
Testing Techniques: Functional, Smoke, Cross Browser, End-to-End, and UI Testing.
